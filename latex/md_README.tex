This repository contains 2 parts\+:
\begin{DoxyItemize}
\item C++ library for GPS and Accelerometer data "{}fusion"{} with Kalman filter.
\item Generator/visualizer -\/ a GTK-\/based application to test this filter
\end{DoxyItemize}

\begin{DoxyItemize}
\item Rewrote filter using C++ from scratch
\item All matrix operations are implemented with \href{https://eigen.tuxfamily.org/}{\texttt{ libeigen}}
\item All geographic operations, distance calculations, etc., are made with \href{https://geographiclib.sourceforge.io/}{\texttt{ Geographic\+Lib}}
\item Changed process noise matrix (Q). See the math behind it \href{https://github.com/rlabbe/Kalman-and-Bayesian-Filters-in-Python/blob/master/07-Kalman-Filter-Math.ipynb}{\texttt{ here}}
\item Fixed bug related to {\ttfamily estimate covariance matrix} (P)
\item Changed the method of how ENU (east-\/north-\/up) acceleration is calculated. Migrated from the rotation matrix to quaternions
\end{DoxyItemize}\hypertarget{md_README_autotoc_md2}{}\doxysection{\texorpdfstring{Requirements}{Requirements}}\label{md_README_autotoc_md2}

\begin{DoxyItemize}
\item Build ideal route (Ctrl + mouse left click on a map) 
\item Set the noise level for the GPS and accelerometer and push the "{}generate"{} button in "{}\+Generator settings"{} section 
\item Set sigmas for MLM filter and push the "{}generate"{} button in "{}\+MLM filter"{} section  ~\newline

\item After this, there will be three layers available\+: ideal data, generated (noisy) data, and filtered route. 
\end{DoxyItemize}\hypertarget{md_README_autotoc_md7}{}\doxysection{\texorpdfstring{How to use filter}{How to use filter}}\label{md_README_autotoc_md7}
See {\ttfamily filter/mlm.\+h} and {\ttfamily filter/sensor\+\_\+data.\+h} for details. But overall, the process looks like this\+:
\begin{DoxyEnumerate}
\item Create the MLM object {\ttfamily mlm}.
\item When the accelerometer vector in ENU is ready, feed it to {\ttfamily mlm} ({\ttfamily bool process\+\_\+acc\+\_\+data(const enu\+\_\+accelerometer \&acc, double time\+\_\+sec);})
\item When the GPS coordinate is ready, feed it to {\ttfamily mlm} ({\ttfamily void process\+\_\+gps\+\_\+data(const gps\+\_\+coordinate \&gps, double time\+\_\+sec);})
\item After receiving each GPS coordinate, get the filtered coordinate from {\ttfamily mlm} ({\ttfamily gps\+\_\+coordinate predicted\+\_\+coordinate() const;})
\end{DoxyEnumerate}

An example of all steps described above can be found \href{https://github.com/maddevsio/mad-location-manager-lib/blob/0eccabb8eeadc7d4c2233d1b24285d8a879c68e7/src/main_window.cpp\#L906C6-L906C40}{\texttt{ here}} An example of how to build Android is in \href{https://github.com/maddevsio/mad-location-manager-lib/wiki/Android}{\texttt{ wiki}}\hypertarget{md_README_autotoc_md8}{}\doxysection{\texorpdfstring{The roadmap}{The roadmap}}\label{md_README_autotoc_md8}
